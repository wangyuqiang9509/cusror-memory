# 测试覆盖率

分析测试覆盖率并补充缺失的测试用例：

1. 执行带覆盖率统计的测试：npm test --coverage 或 pnpm test --coverage

2. 解析覆盖率报告（coverage/coverage-summary.json）

3. 筛选覆盖率低于 80% 阈值的文件

4. 针对每个覆盖不足的文件：
   - 分析未覆盖的代码路径
   - 为函数生成单元测试
   - 为 API 生成集成测试
   - 为关键流程生成端到端（E2E）测试

5. 验证新增测试全部通过

6. 展示覆盖率前后对比数据

7. 确保项目整体覆盖率达到 80% 以上

重点关注：
- 正常路径场景（Happy Path）
- 错误处理逻辑
- 边界情况（null、undefined、空值）
- 临界条件
