# 验证命令

对当前代码库状态执行全面验证。

## 指令

按以下顺序依次执行验证：

1. **构建检查**
   - 运行项目的构建命令
   - 若构建失败，报告错误并**立即终止**

2. **类型检查**
   - 运行 TypeScript / 类型检查器
   - 报告所有错误，格式：`文件:行号`

3. **代码规范检查**
   - 运行 Linter
   - 报告警告与错误

4. **测试套件**
   - 运行全部测试
   - 报告通过/失败数量
   - 报告覆盖率百分比

5. **Console.log 审计**
   - 在源代码中搜索 `console.log`
   - 报告所有出现位置

6. **Git 状态**
   - 显示未提交的变更
   - 显示自上次提交以来修改的文件

## 输出格式

生成简洁的验证报告：

```
验证结果: [通过/失败]

构建:     [OK/失败]
类型:     [OK/X 个错误]
规范:     [OK/X 个问题]
测试:     [X/Y 通过, Z% 覆盖率]
密钥泄露: [OK/发现 X 处]
日志:     [OK/X 处 console.log]

可提交 PR: [是/否]
```

若存在关键问题，列出问题并附修复建议。

## 参数

$ARGUMENTS 可选值：
- `quick` — 仅执行构建 + 类型检查
- `full` — 执行全部检查（默认）
- `pre-commit` — 执行与提交相关的检查
- `pre-pr` — 执行全部检查 + 安全扫描
