# 代码审查

对未提交变更进行全面的安全与质量审查：

1. 获取变更文件列表：git diff --name-only HEAD

2. 逐一检查每个变更文件，排查以下问题：

**安全问题（严重 CRITICAL）：**
- 硬编码的凭证、API 密钥、令牌
- SQL 注入漏洞
- XSS 跨站脚本漏洞
- 缺失输入验证
- 不安全的依赖项
- 路径遍历风险

**代码质量（高 HIGH）：**
- 函数超过 50 行
- 文件超过 800 行
- 嵌套层级超过 4 层
- 缺失错误处理
- 残留 console.log 语句
- TODO/FIXME 注释未清理
- 公共 API 缺少 JSDoc 文档

**最佳实践（中 MEDIUM）：**
- 存在可变模式（应使用不可变模式）
- 代码或注释中使用表情符号
- 新增代码缺少测试覆盖
- 无障碍访问问题（a11y）

3. 生成审查报告，包含：
   - 严重等级：CRITICAL、HIGH、MEDIUM、LOW
   - 文件路径及行号
   - 问题描述
   - 修复建议

4. 若存在 CRITICAL 或 HIGH 级别问题，阻止提交

绝不放行存在安全漏洞的代码！
