# Eval 命令

管理评估驱动开发（Eval-Driven Development）工作流。

## 用法

`/eval [define|check|report|list] [功能名称]`

## 定义评估

`/eval define 功能名称`

创建新的评估定义：

1. 使用以下模板创建 `.claude/evals/功能名称.md`：

```markdown
## EVAL: 功能名称
创建时间: $(date)

### 能力评估
- [ ] [能力描述 1]
- [ ] [能力描述 2]

### 回归评估
- [ ] [现有行为 1 仍正常工作]
- [ ] [现有行为 2 仍正常工作]

### 成功标准
- 能力评估 pass@3 > 90%
- 回归评估 pass^3 = 100%
```

2. 引导用户填写具体评估标准

## 检查评估

`/eval check 功能名称`

运行指定功能的评估：

1. 从 `.claude/evals/功能名称.md` 读取评估定义
2. 对每项能力评估：
   - 尝试验证评估标准
   - 记录 PASS/FAIL
   - 将尝试结果记录到 `.claude/evals/功能名称.log`
3. 对每项回归评估：
   - 运行相关测试
   - 与基线进行对比
   - 记录 PASS/FAIL
4. 报告当前状态：

```
评估检查: 功能名称
========================
能力评估: X/Y 通过
回归评估: X/Y 通过
状态: 进行中 / 就绪
```

## 评估报告

`/eval report 功能名称`

生成完整评估报告：

```
评估报告: 功能名称
=========================
生成时间: $(date)

能力评估
----------------
[eval-1]: PASS (pass@1)
[eval-2]: PASS (pass@2) - 需要重试
[eval-3]: FAIL - 见备注

回归评估
----------------
[test-1]: PASS
[test-2]: PASS
[test-3]: PASS

指标
-------
能力评估 pass@1: 67%
能力评估 pass@3: 100%
回归评估 pass^3: 100%

备注
-----
[问题、边缘情况或观察记录]

建议
--------------
[发布 / 需改进 / 已阻塞]
```

## 列出评估

`/eval list`

显示所有评估定义：

```
评估定义
================
feature-auth      [3/5 通过] 进行中
feature-search    [5/5 通过] 就绪
feature-export    [0/4 通过] 未开始
```

## 参数

$ARGUMENTS:
- `define <名称>` - 创建新的评估定义
- `check <名称>` - 运行并检查评估
- `report <名称>` - 生成完整报告
- `list` - 显示所有评估
- `clean` - 清理旧评估日志（保留最近 10 次运行记录）
