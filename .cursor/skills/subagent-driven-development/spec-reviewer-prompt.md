# 规格符合性审查者提示模板

在分派规格符合性审查者子代理时使用此模板。

**目的：** 验证实现者是否构建了被要求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规格符合性"
  prompt: |
    你正在审查实现是否符合其规格。

    ## 被要求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 重要：不要相信报告

    实现者完成得可疑地快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们说实现了什么
    - 相信他们关于完整性的声称
    - 接受他们对需求的解释

    **要：**
    - 阅读他们实际编写的代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被要求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某功能有效但实际没有实现？

    **多余/不需要的工作：**
    - 他们是否构建了未被要求的内容？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规格中没有的"锦上添花"功能？

    **误解：**
    - 他们对需求的理解是否与预期不同？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但用了错误的方式？

    **通过阅读代码验证，而非相信报告。**

    汇报：
    - ✅ 符合规格（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附带 文件:行号 引用]
```
