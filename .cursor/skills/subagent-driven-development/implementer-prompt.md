# 实现者子代理提示模板

在分派实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N: [任务名称]"
  prompt: |
    你正在实现任务 N: [任务名称]

    ## 任务描述

    [任务的完整文本 - 粘贴在这里，不要让子代理读取文件]

    ## 上下文

    [场景设定：任务定位、依赖关系、架构上下文]

    ## 开始之前

    如果你对以下方面有疑问：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在就问。** 在开始工作之前提出任何疑虑。

    ## 你的工作

    一旦你清楚了需求：
    1. 完全按照任务规格实现
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实现是否有效
    4. 提交你的工作
    5. 自审（见下文）
    6. 汇报

    工作目录: [目录]

    **工作过程中：** 如果遇到意外或不清楚的情况，**提问**。
    随时可以暂停并澄清。不要猜测或做假设。

    ## 汇报前：自审

    用全新的眼光审查你的工作。问问自己：

    **完整性：**
    - 我是否完全实现了规格中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我没有处理的边界情况？

    **质量：**
    - 这是我最好的作品吗？
    - 命名是否清晰准确（匹配功能而非实现方式）？
    - 代码是否整洁、可维护？

    **纪律性：**
    - 我是否避免了过度构建（YAGNI）？
    - 我是否只构建了被要求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而非仅仅模拟行为）？
    - 如果需要，我是否遵循了 TDD？
    - 测试是否全面？

    如果在自审中发现问题，现在就修复，然后再汇报。

    ## 汇报格式

    完成后，汇报：
    - 你实现了什么
    - 你测试了什么以及测试结果
    - 更改的文件
    - 自审发现（如有）
    - 任何问题或疑虑
```
