# 执行计划

## 概述

加载计划、进行关键性审查、按批次执行任务，并在批次之间提交报告以供审查。

**核心原则：** 批量执行，并在关键节点由架构师进行审查。

**启动时声明：**“我正在使用执行计划技能来实施该计划。”

## 流程

### 步骤 1：加载并审查计划
1. 读取计划文件
2. 进行关键性审查——识别计划中的任何问题或疑虑
3. 如有疑虑：在开始之前向你的人类搭档提出
4. 如无疑虑：创建 TodoWrite 并继续

### 步骤 2：执行批次
**默认：前 3 个任务**

对于每个任务：
1. 标记为进行中（in_progress）
2. 严格按照每一步执行（计划中的步骤应易于操作）
3. 运行指定的验证
4. 标记为已完成（completed）

### 步骤 3：报告
当批次完成时：
- 展示已实现的内容
- 展示验证输出
- 说：“准备接收反馈。”

### 步骤 4：继续
根据反馈：
- 如有需要，应用更改
- 执行下一批次
- 重复直到完成

### 步骤 5：完成开发

在所有任务完成并通过验证后：
- 宣布：“我正在使用完成开发分支技能来完成这项工作。”
- **所需子技能：** 使用 finishing-a-development-branch skill
- 按照该技能验证测试、展示选项并执行选择

## 何时停止并寻求帮助

**遇到以下情况时立即停止执行：**
- 在批次中遇到阻碍（缺少依赖、测试失败、指令不明确）
- 计划存在关键缺陷，导致无法启动
- 你不理解某条指令
- 验证多次失败

**如有疑问，请求澄清，而不是猜测。**

## 何时回顾之前的步骤

**在以下情况下返回审查（步骤 1）：**
- 搭档根据你的反馈更新了计划
- 需要重新思考基本方法

**不要强行突破阻碍**——停下来询问。

## 记住
- 首先对计划进行关键性审查
- 严格按照计划步骤执行
- 不要跳过验证
- 当计划提到某项技能时，引用该技能
- 批次之间：仅报告并等待
- 遇到阻碍时停止，不要猜测
